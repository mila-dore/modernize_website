Guide Drupal CMS → Drupal Headless + Next.js

1. Architecture cible

Drupal
  ↓
JSON:API
  ↓
Next.js
  ↓
Utilisateur

Drupal gère le contenu. Next.js gère l'affichage.

2. Pourquoi JSON:API ?

Ne pas accéder directement aux tables MySQL/PostgreSQL.

  • Sécurité
  • Permissions Drupal
  • Workflows
  • Structure interne masquée
  • Maintenance simplifiée

6. Passage à Drupal

export async function getArticles() {
  const res = await fetch(
    `${process.env.DRUPAL_URL}/jsonapi/node/article`
  )

  const json = await res.json()
  return json.data
}

7. Cache et revalidation

Toujours frais

fetch(url, {
  cache: "no-store"
})

Revalidation

fetch(url, {
  next: {
    revalidate: 60
  }
})
Quand un contenu est créé dans Drupal, JSON:API le voit immédiatement. Le délai éventuel vient généralement du cache Next.js.
